# 音乐播放器前端界面元素设计

本文档详细描述了 `myMusicPlayer-rs` 项目的前端界面（GUI）核心元素、布局和组件设计，旨在为Svelte前端开发提供清晰的指引。

## 1. 总体布局 (Main Layout)

应用采用经典的三区布局，确保在不同尺寸的桌面屏幕上都有一致且高效的体验。

```
+------------------------------------------------------+
| Top Bar (全局搜索、窗口控制)                         |
+----------------------+-------------------------------+
|                      |                               |
|                      |                               |
| Left Sidebar         | Main Content                  |
| (导航)               | (内容展示区)                  |
|                      |                               |
|                      |                               |
|                      |                               |
+----------------------+-------------------------------+
| Bottom Player Bar (播放控制条)                       |
+------------------------------------------------------+
```

- **顶部栏 (Top Bar)**: (可选，可与主内容区顶部合并) 提供全局搜索功能和窗口控制按钮（最小化、最大化、关闭）。
- **左侧边栏 (Left Sidebar)**: 核心导航区域。
- **主内容区 (Main Content)**: 根据侧边栏的导航选择，动态显示不同内容。
- **底部播放器 (Bottom Player Bar)**: 始终固定在底部，提供全局的播放控制。

## 2. 核心组件设计

### 2.1 底部播放器 (Bottom Player Bar)

这是应用交互最频繁的区域。从左到右分为三个部分：

#### 2.1.1 左侧：当前歌曲信息
- **专辑封面 (Album Art)**: 一个小尺寸的方形缩略图，点击可放大或跳转至专辑详情页。
- **歌曲信息 (Track Info)**:
    - **歌曲标题 (Track Title)**: 字体加粗，显示当前播放的歌曲名。
    - **艺术家 (Artist Name)**: 灰色或次要颜色字体，显示艺术家名。
- **收藏按钮 (Favorite Button)**: 一个“心形”图标按钮，用于将歌曲添加到“我喜欢的音乐”列表。

#### 2.1.2 中间：播放控制
- **播放进度条 (Progress Bar)**:
    - 一条可拖拽和点击的进度条，用于调整播放进度 (Seek)。
    - 左侧显示当前播放时间（如 `1:23`）。
    - 右侧显示歌曲总时长（如 `3:45`）。
- **核心控制按钮 (Main Controls)**:
    - **随机播放 (Shuffle Button)**: 切换播放队列的随机模式。
    - **上一首 (Previous Button)**: 跳转到上一首歌曲。
    - **播放/暂停 (Play/Pause Button)**: 核心按钮，尺寸稍大，用于控制播放和暂停。
    - **下一首 (Next Button)**: 跳转到下一首歌曲。
    - **循环模式 (Repeat Button)**: 切换循环模式（不循环、列表循环、单曲循环）。

#### 2.1.3 右侧：辅助功能
- **播放队列 (Queue Button)**: 点击打开一个弹窗或侧边栏，显示即将播放的歌曲列表。
- **音量控制 (Volume Control)**:
    - 一个“扬声器”图标。
    - 鼠标悬停或点击时，显示一个垂直或水平的音量调节滑块。
- **(可选) 歌词按钮 (Lyrics Button)**: 点击可打开一个覆盖层或跳转到专门的歌词显示界面。
- **(可选) 音频输出设备 (Device Picker)**: 点击可选择不同的音频输出设备。

### 2.2 左侧边栏 (Left Sidebar)

提供清晰的导航结构。

- **主导航 (Main Navigation)**:
    - **首页 (Home)**: (未来功能) 应用的仪表盘或推荐页。
    - **音乐库 (Library)**: 展开或跳转至用户的个人音乐收藏。
    - **设置 (Settings)**: 跳转到应用设置页面。
- **音乐库子导航 (Library Sub-navigation)**:
    - **歌曲 (Songs)**: 显示所有歌曲的列表。
    - **专辑 (Albums)**: 以网格视图显示所有专辑。
    - **艺术家 (Artists)**: 显示所有艺术家的列表。
- **播放列表 (Playlists)**:
    - 一个可滚动的列表，显示用户创建的所有播放列表。
    - 列表底部有“创建播放列表”按钮。

### 2.3 主内容区 (Main Content)

根据导航选择显示不同的视图。

#### 2.3.1 歌曲视图 (Songs View)
- 一个可排序的表格，用于展示所有歌曲。
- **表头列**:
    - `#` (索引)
    - `标题` (包含歌曲名和艺术家)
    - `专辑`
    - `时长`
    - `添加时间`
- **交互**:
    - 双击行项目可直接播放该歌曲。
    - 右键单击行项目可打开上下文菜单（如：添加到播放列表、从音乐库删除等）。

#### 2.3.2 专辑视图 (Albums View)
- 一个响应式的网格布局，展示所有专辑的封面。
- 每个网格项包含：
    - **专辑封面**
    - **专辑标题**
    - **艺术家**
- **交互**: 点击任意专辑，主内容区切换到该专辑的“专辑详情视图”。

#### 2.3.3 艺术家视图 (Artists View)
- 一个列表或网格，展示所有艺术家的圆形头像或姓名。
- **交互**: 点击任意艺术家，主内容区切换到该艺术家的“艺术家详情视图”。

#### 2.3.4 专辑/播放列表详情视图 (Album/Playlist Detail View)
- **顶部横幅 (Header)**:
    - 左侧为较大的专辑封面或播放列表封面。
    - 右侧为详细信息：
        - 类型 (专辑/播放列表)
        - 标题
        - 创作者/艺术家
        - 简介、年份、总时长、歌曲数量等。
    - 一个醒目的“播放”按钮，用于播放整个专辑或列表。
- **歌曲列表 (Track List)**:
    - 与“歌曲视图”类似的表格，但只包含当前专辑或播放列表中的歌曲。
    - 列包含：`#` (音轨号), `标题`, `时长`。

#### 2.3.5 设置视图 (Settings View)
- 使用分组和标签来清晰地组织各项配置。
- **音乐库 (Library)**:
    - 显示当前扫描的音乐文件夹路径列表。
    - “添加文件夹”按钮。
    - “立即重新扫描”按钮。
- **界面 (Appearance)**:
    - **主题 (Theme)**: 提供“亮色”、“暗色”、“跟随系统”三个选项。
- **音频 (Audio)**:
    - **输出设备 (Output Device)**: 一个下拉菜单，列出所有可用的音频输出设备。
- **关于 (About)**:
    - 显示应用名称、版本号、作者和开源许可信息。
